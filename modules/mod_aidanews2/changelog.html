Version 2.0.0 - First AiDaNews2 release

What's new compared to AiDaNews 1.2.1?

[+] Secondary Ordering option - select how to order the articles when you can't discern an order from the Primary Ordering (for example if they have the same number of views or comments).
[+] Three separate images can now be displayed for every article. They can be article images, user avatars, youtube videos thumbnails, category images and so on. You can also link them to separate urls.
[+] Images Preference list - if an image isn't available for the article, the module will look for the next one in the list.
[+] First / Last Image - You can now choose to select the last image tag, thumbnail, gallery (...) instead of the first one. If you put both First and Last of the same image type, the module will automatically remove the second in the list (as there won't be last if there isn't first :3)
[+] New Layout Positions - you now have 9 fields instead of 5, and they can be styled in different ways to adapt them to your needs.
[+] CSS files support - choose wheter to use the default CSS style, a pre-made one that suits your template best or create your own. You can still add some/all of the CSS rules you need from the module's backend!
[+] Layout Styles - if you chose to use a CSS file, you can use its own module positions that are stored in the layout file with the same name. More info about this can be found in the Documentation ;).

[-] FLEXIcontent support has been removed from the main module. A new module for the component may be released in the future, but it will be separate from this one.
[-] The option that allowed to exclude categories has not been recoded for this new version. I'll include it in a new version if anyone needs it - just let me know with an e-mail or a post on the site. The same reasoning applies to Facebook and Twitter share links, rating with stars, cateogry display and the author limit to JomSocial and CB friends.
